# ジェスチャー描画メモ

## 描画方式

- WPF の透明オーバーレイウィンドウを常駐させ、`Canvas` 上に軌跡を描画する方式
- 入力中は 2 層の `Polyline` で線を更新し、現在位置は `Ellipse` で補助表示する
- マウスフック側は描画自体を持たず、ポイント列だけをイベントで通知する

## 描画レイヤーの構成

- `MouseHookService`
  - 低レベルマウスフック
  - 入力開始 / 移動 / 終了のたびに軌跡イベントを発火
- `GestureTrailOverlayService`
  - UI スレッド上で描画イベントを受ける
  - 軌跡の表示、更新、クリアタイマー制御を担当
- `GestureOverlayWindow`
  - 仮想スクリーン全体を覆う透明ウィンドウ
  - `WS_EX_TRANSPARENT | WS_EX_TOOLWINDOW | WS_EX_NOACTIVATE` を付与
  - 通常操作の邪魔をしないクリック透過レイヤー

## Trigger A/B の色分け仕様

- Trigger A: 赤
- Trigger B: 青
- 現状は固定色
- 実装上は `GestureTrailOverlayService` 内で TriggerSlot から Brush を選択しているため、将来の色設定追加はこの層を広げれば対応しやすい
- 見た目調整として、純色 1 本線ではなく
  - 内側: くっきりした本線
  - 外側: 半透明の太いグロー
  の 2 層構成にした

## 入力開始から終了までの表示ルール

1. 開始ボタン押下
   - 軌跡表示を開始
   - 最初の 1 点だけでも現在位置ドットを表示
2. 入力中
   - `WM_MOUSEMOVE` ごとにポイント列を更新
   - 線と現在位置ドットをリアルタイム更新
   - 現在位置ドットは白縁付きにして背景上でも見失いにくくした
3. 入力終了
   - ボタン離し時点のポイント列で最終描画
   - 既定では `450ms` 残してから自動消去
4. 短いクリック
   - クリック再生は行うが、軌跡イベントは終了まで流す
   - そのため入力取得確認用の表示は行われる

## 今後の拡張ポイント

- UI からの色変更
- UI からの線幅変更
- フェードアウト時間設定
- モニタごとの DPI を意識した描画補正
- 軌跡を直線ではなく滑らかな `PathGeometry` にする改善
- 認識済み / 未認識で色や表示パターンを変える拡張
- 入力無効時の描画抑止オプション
