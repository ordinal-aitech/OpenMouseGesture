# ジェスチャーアクションラベルオーバーレイ

## 修正前の状態
- OpenMouseGesture には軌跡表示はあるが、認識済みアクション名や Hotkey を入力中に見せる補助表示は無かった
- ジェスチャー結果は action 実行後にしか体感しづらく、実行前の予告表示は無かった

## 表示仕様
- ジェスチャーが認識され、かつ対応する action が解決できた時点で表示する
- トリガーボタンを押している間は表示を維持する
- トリガーボタンを離したら消す
- 未認識状態では表示しない
- 入力中に認識候補が変わった場合は表示内容を更新する

## 表示開始条件
- ドラッグ中
- 一定数以上の軌跡点がある
- ジェスチャー認識が成功する
- その gesture に紐づく action が見つかる

## 表示終了条件
- ボタン解放
- 認識不能になった場合
- 認識済みでも action が見つからない場合

## 表示内容の決定ルール
- 優先:
  - action 名
- 補助:
  - 実際の操作内容

### 例
- action 名がある場合:
  - 1行目: action 名
  - 2行目: `Ctrl+C` などの実操作
- action 名が無い場合:
  - 1行目: アクション種別
  - 2行目: 実操作

### 実装済みの detail 生成
- keystroke:
  - `Ctrl+C`
  - `Alt+Left`
- window operation:
  - `最小化`
  - `最大化`
  - `閉じる`
- command:
  - コマンド文字列
- url:
  - URL 文字列

## 実装方式

### 採用方式
- 軌跡オーバーレイとは別の軽量オーバーレイを追加

### 判断理由
- 軌跡オーバーレイは軌跡周辺だけを小さく更新する設計
- ラベルは画面下部に固定表示したい
- 同一ウィンドウに載せると、軌跡とラベルの更新範囲をまとめて扱う必要があり、描画範囲が大きくなりやすい
- 別オーバーレイにした方が、軌跡描画と補助表示を独立に制御しやすい

### 追加した構成
- `src-tauri/src/action_label_overlay.rs`
  - 角丸ラベル描画
  - 白文字描画
  - 画面下部中央寄りの layered window 表示
- `src-tauri/src/lib.rs`
  - action 表示文言の整形
  - オーバーレイ表示 / 消去のラッパー
- `src-tauri/src/mouse_hook.rs`
  - ドラッグ中プレビュー認識
  - 表示開始 / 更新 / 終了の状態連動

## 見た目
- 小さなオレンジ系ラベル
- 角丸
- 白文字
- 2 行まで表示可能
- 画面下部中央寄りに一時表示

## 2026-06-28 追記

### 今回の不具合内容
- ユーザー実機で、action label overlay を有効にした状態だと以下の重大な不具合が発生
  - マウスカーソルがスローモーションになる
  - 軌跡が残ったままになる
  - マウスジェスチャーソフト自体がフリーズする

### 一旦無効化した理由
- ラベル表示の見栄えより、通常のマウスジェスチャー操作を壊さないことを優先するため
- 実機で操作感と安定性に明確な悪影響が出ていたため、今回は「改善して維持」ではなく「停止 / 無効化」を優先した

### 表示されなかった原因
- 実機確認結果として「画面下部ラベルが出ていない」ことが確認された
- コード調査で重なっていた可能性が高い点:
  - 認識プレビュー開始条件が遅めだった
  - 同じ認識結果では再送が弱かった
  - overlay 更新が window message 頼みで、初期化や message queue のタイミングに依存していた
  - 表示サイズと下端マージンが小さく、見逃しやすい可能性があった

### 修正内容
- `PREVIEW_MIN_POINTS` を `6` に引き下げ
- `PREVIEW_INTERVAL_MS` を `16ms` に短縮
- 認識済み action が継続している間は、同じ action でもラベル更新を再送するよう変更
- ButtonUp 処理に入る直前にも `update_recognition_preview(true)` を呼び、最終区間でも表示判定が走るようにした
- `action_label_overlay.rs` 側は、`PostMessageW` 経由の更新から、`update_action_label()` 呼び出し時にその場で layered window を反映する構成へ変更
- overlay の準備完了フラグを追加し、初期化待ちを `sleep` 依存から改善
- overlay サイズを拡大:
  - `320x84 -> 420x108`
- 画面下端マージンを拡大:
  - `48px -> 88px`
- `SetWindowPos(..., HWND_TOPMOST, ..., SWP_SHOWWINDOW)` を追加し、前面表示を強めた
- debug 用に `show / hide` の `eprintln!` を追加

### 無効化の実装方法
- `lib.rs` に `ACTION_LABEL_OVERLAY_ENABLED = false` を追加
- 起動時の `action_label_overlay::init_overlay()` 呼び出しを flag で停止
- `show_action_label_for_action()` を no-op 化
- `clear_action_label_overlay()` も no-op 化
- これにより overlay 用 window / 描画更新 / clear 処理が通常の gesture 経路へ介入しない状態に戻した

### 表示開始条件
- trigger ボタン押下中
- 軌跡点数が最小条件以上
- gesture 認識が成功
- `trigger_slot + gesture` に対応する action が見つかる

### 表示終了条件
- trigger ボタン解放
- 認識不能になった場合
- 認識済みでも action が見つからない場合

### 実機確認結果
- 自動確認:
  - ビルド成功
  - アプリ起動成功
  - overlay 無効化後の release build 成功
- 物理マウス実機確認:
  - 修正前: 「表示されていない」ことをユーザー実機で確認済み
  - 修正前の追加症状として、カーソル低速化・軌跡残り・フリーズが確認済み
  - 修正後: こちらの環境では未確認

### 未確認事項
- 無効化後に、物理マウスで重さ・フリーズ・軌跡残りが解消したか
- 再導入時の、低負荷な表示方式

### 再導入時の注意点
- overlay 更新を低レベルマウスフック内の重い処理に寄せない
- gesture 認識と UI 表示は可能なら非同期に切り離す
- layered window の生成・更新・破棄タイミングが drag 中の入力処理へ干渉しない構成にする
- 軌跡 renderer と action label renderer の責務を明確に分離する
